db.goal_repo module¶

class src.db.goal_repo.GoalRepo(con)¶

Bases: object

Methods to manipulate “goal” table in the Goalz database

The sqlite3 connection instance is received as a constructor parameter and is accessible to all the methods of this class through the self.con attribute.

Methods of this class MUST not be accessed directly. All the calls to the database should be made through the API provided by Connection

Parameters:con (sqlite3.Connection) – Connection to an SqlLite database

get_goals(user_id, number_of_goals, before, after)¶

Return a list of all the goals in the database filtered by the conditions provided in the parameters.

Parameters:
  • user_id (int) – Search goals of a user with the given user_id. If this parameter is None, it returns the goals of any user in the system.
  • number_of_goals (int) – Sets the maximum number of goals returning in the list. If set to None, there is no limit.
  • before (long) – All deadlines > before (UNIX timestamp) are removed. If set to None, this condition is not applied.
  • after (long) – All deadlines < after (UNIX timestamp) are removed. If set to None, this condition is not applied.
Returns:

A list of goals. Each goal is a dictionary containing the following keys:

  • goal_id: integer representing the Id of the goal..
  • title: string containing the title of the goal.
  • description: string containing the description of goal.

Note that all values in the returned dictionary are string unless otherwise stated.
Raises:

ValueError – if before or after are not valid UNIX timestamps
